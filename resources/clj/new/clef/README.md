# {{title}}

A web application generated by [clef](https://github.com/kirahowe/clef).
It boots an [Integrant](https://github.com/weavejester/integrant)-managed
system — an http-kit server, Reitit routing, Hiccup-rendered HTML, and
Telemere logging — configured entirely through layered EDN profiles.

## Prerequisites

- [Clojure CLI](https://clojure.org/guides/install_clojure) (Java 21+)

## Run it locally

Start a REPL with the `:dev` alias:

```
clj -M:dev
```

Then bring the system up:

```clojure
(dev)   ;; load the dev namespace + integrant.repl helpers
(go)    ;; start the system
(reset) ;; reload changed code and restart
(halt)  ;; stop the system
```

The app will be available at http://localhost:3000 (the port set in
`env/dev/resources/dev.edn`).

## Tasks

| Command                 | What it does                          |
| ----------------------- | ------------------------------------- |
| `clj -M:test`           | run the test suite (Kaocha)           |
| `clj -M:lint`           | clj-kondo over `src`, `test`, `env`   |
| `clj -M:cljfmt check`   | check formatting (cljfmt)             |
| `clj -M:prod`           | run with the production profile       |

## Configuration

Configuration is EDN, not scattered env vars. `resources/base-system.edn`
is always loaded; the profile you name on the command line is meta-merged
on top:

```
resources/base-system.edn      # defaults, always loaded
env/dev/resources/dev.edn      # (dev) / (go) in the REPL
env/test/resources/test.edn    # clj -M:test
env/prod/resources/prod.edn    # clj -M:prod
```

The few values that must come from the environment (ports, secrets) are
pulled in inline via reader literals like `#env/long ["PORT" 8080]`.
